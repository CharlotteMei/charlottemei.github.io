Test Framework Scaling by Containerization

Project Motivation:
Previously, the testing phase of the development cycle takes too long, and one of the bottlenecks is that all the tests in the same batch would be run in the same machine.

Solution and Implementation:
I have designed and implemented a CLI application which binds with the testing frameworks, so that the tests can be indivisually containerized and run on a cluster of hosts.

Main Technology & Tools Used:
Docker, AWS ECS and ECR, and some other internal orchestrating API tools. The project was implemented in Python

Team: AWS Network DevX (prev. SALT)
